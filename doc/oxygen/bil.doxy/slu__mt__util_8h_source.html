149 /*
150  * *********************************************************************
151  * The superlumt_options_t structure contains the shared variables used
152  * for factorization, which are passed to each thread.
153  * *********************************************************************
154  *
155  * nprocs (int)
156  * Number of processes (or threads) to be spawned and used to perform
157  * the LU factorization by pdgstrf().
158  *
159  * fact (fact_t)
160  * Specifies whether or not the factored form of the matrix
161  * A is supplied on entry, and if not, whether the matrix A should
162  * be equilibrated before it is factored.
163  * = DOFACT: The matrix A will be factored, and the factors will be
164  * stored in L and U.
165  * = EQUILIBRATE: The matrix A will be equilibrated if necessary, then
166  * factored into L and U.
167  *
168  * trans (trans_t)
169  * Specifies the form of the system of equations:
170  * = NOTRANS: A * X = B (No transpose)
171  * = TRANS: A**T * X = B (Transpose)
172  * = CONJ: A**H * X = B (Transpose)
173  *
174  * refact (yes_no_t)
175  * Specifies whether this is first time or subsequent factorization.
176  * = NO: this factorization is treated as the first one;
177  * = YES: it means that a factorization was performed prior to this
178  * one. Therefore, this factorization will re-use some
179  * existing data structures, such as L and U storage, column
180  * elimination tree, and the symbolic information of the
181  * Householder matrix.
182  *
183  * panel_size (int)
184  * A panel consists of at most panel_size consecutive columns.
185  *
186  * relax (int)
187  * To control degree of relaxing supernodes. If the number
188  * of nodes (columns) in a subtree of the elimination tree is less
189  * than relax, this subtree is considered as one supernode,
190  * regardless of the row structures of those columns.
191  *
192  * diag_pivot_thresh (double)
193  * Diagonal pivoting threshold. At step j of the Gaussian elimination,
194  * if abs(A_jj) >= diag_pivot_thresh * (max_(i>=j) abs(A_ij)),
195  * use A_jj as pivot, else use A_ij with maximum magnitude.
196  * 0 <= diag_pivot_thresh <= 1. The default value is 1,
197  * corresponding to partial pivoting.
198  *
199  * drop_tol (double) (NOT IMPLEMENTED)
200  * Drop tolerance parameter. At step j of the Gaussian elimination,
201  * if abs(A_ij)/(max_i abs(A_ij)) < drop_tol, drop entry A_ij.
202  * 0 <= drop_tol <= 1. The default value of drop_tol is 0,
203  * corresponding to not dropping any entry.
204  *
205  * usepr (yes_no_t)
206  * Whether the pivoting will use perm_r specified by the user.
207  * = YES: use perm_r; perm_r is input, unchanged on exit.
208  * = NO: perm_r is determined by partial pivoting, and is output.
209  *
210  * SymmetricMode (yest_no_t)
211  * Specifies whether to use symmetric mode.
212  *
213  * PrintStat (yes_no_t)
214  * Specifies whether to print solver's statistics.
215  *
216  * perm_c (int*) dimension A->ncol
217  * Column permutation vector, which defines the
218  * permutation matrix Pc; perm_c[i] = j means column i of A is
219  * in position j in A*Pc.
220  * When search for diagonal, perm_c[*] is applied to the
221  * row subscripts of A, so that diagonal threshold pivoting
222  * can find the diagonal of A, instead of that of A*Pc.
223  *
224  * perm_r (int*) dimension A->nrow
225  * Row permutation vector which defines the permutation matrix Pr,
226  * perm_r[i] = j means row i of A is in position j in Pr*A.
227  * If usepr = NO, perm_r is output argument;
228  * If usepr = YES, the pivoting routine will try to use the input
229  * perm_r, unless a certain threshold criterion is violated.
230  * In that case, perm_r is overwritten by a new permutation
231  * determined by partial pivoting or diagonal threshold pivoting.
232  *
233  * work (void*) of size lwork
234  * User-supplied work space and space for the output data structures.
235  * Not referenced if lwork = 0;
236  *
237  * lwork (int)
238  * Specifies the length of work array.
239  * = 0: allocate space internally by system malloc;
240  * > 0: use user-supplied work array of length lwork in bytes,
241  * returns error if space runs out.
242  * = -1: the routine guesses the amount of space needed without
243  * performing the factorization, and returns it in
244  * superlu_memusage->total_needed; no other side effects.
245  *
246  * etree (int*)
247  * Elimination tree of A'*A, dimension A->ncol.
248  * Note: etree is a vector of parent pointers for a forest whose
249  * vertices are the integers 0 to A->ncol-1; etree[root]==A->ncol.
250  * On input, the columns of A should be permutated so that the
251  * etree is in a certain postorder.
252  *
253  * colcnt_h (int*)
254  * Column colunts of the Householder matrix.
255  *
256  * part_super_h (int*)
257  * Partition of the supernodes in the Householder matrix.
258  * part_super_h[k] = size of the supernode beginning at column k;
259  * = 0, elsewhere.
260  *
261  *
262  */
